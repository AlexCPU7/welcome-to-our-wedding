# AGENTS.md

Контекст для следующих сессий по проекту `site-welcome`.

## Проект

- Стек: Vite + React + TypeScript.
- Основная точка входа: `src/App.tsx`.
- Глобальные стили: `src/styles/globals.css`.
- Стили Italy/Limoncello: `src/styles/limoncello.css`.
- Компоненты: `src/components`.
- Данные свадьбы и контент: `src/data/wedding.ts`.
- Варианты дизайна: `src/data/designVariants.ts`.

## Команды проверки

- Основная проверка после правок: `npm run build`.
- В проекте используется TypeScript strict; `npm run build` запускает `tsc -b` и сборку Vite.
- Для визуальных проверок ранее использовались временные Playwright/Vite-скрипты в `.omx/*.mjs`; после проверки их нужно удалять.

## Главный продуктовый контекст

Сайт — свадебное приглашение Алексея и Марины.

Сейчас рабочий и основной публичный дизайн — **Italy / Limoncello** (`limoncello`).

Важно:

- Работать с дизайном Italy как с основным.
- Код дизайна Pearl оставить в проекте, не удалять.
- Без явного запроса пользователя не менять Pearl.
- Не возвращать плашку выбора дизайна.
- Не добавлять `design=limoncello` в URL.
- Ссылки с `uuid` должны открываться сразу в Italy и сохранять `uuid` в URL.

Текущее состояние:

- `src/data/designVariants.ts`: `defaultDesignVariant = 'limoncello'`.
- `src/App.tsx`: `DesignSwitcher` отключён/закомментирован.
- `App.tsx` выставляет `document.documentElement.dataset.design = 'limoncello'`.
- Если в URL есть `design`, он удаляется через `history.replaceState`, но `uuid` остаётся.

## Пользовательские предпочтения по дизайну

- Делать качественно и законченно, особенно на mobile.
- Не оставлять “примерно работает” — проверять сборку и, если возможно, визуально.
- Все блоки в Italy должны выглядеть цельно, отцентрировано и аккуратно.
- Не использовать тяжёлый капс там, где он ухудшает читаемость.
- Текст должен быть чуть крупнее среднего, комфортный для мобильного чтения.
- Не лечить сложные композиции бесконечными пиксельными костылями, если лучше сделать устойчивую структуру.
- Для блока “Тайминг дня” особенно важно сохранять композицию линий и угловых картинок на mobile/desktop.

## Italy / Limoncello: важные текущие решения

### Hero

- В верхнем блоке отображаются “Алексей”, `&`, “Марина”.
- Между именами была синяя линия через `.hero__title span::before`.
- Пользователь попросил убрать видимую линию, но после удаления `&` съехал.
- Сейчас `::before` оставлен как невидимая распорка:
  - `display: block`
  - `background: transparent`
  - `opacity: 0`
  - это сохраняет позицию `&`.
- Не менять это на `display: none`, иначе `&` снова уедет влево.

### Locations / “Где встречаемся”

- Порядок внутри карточки: фото → название → адрес → кнопка Яндекс Карт.
- Для Italy используется фото ресторана `public/images/LaurenParker.webp`.
- Кнопка Яндекс Карт: жёлтый фон, синий текст.
- Фото, заголовок, адрес и кнопка должны быть на одном визуальном центре.
- Для стабильного центрирования добавлены:
  - `.location-card { grid-template-columns: minmax(0, 1fr); }`
  - `justify-self: center` для фото, body и `.map-actions`.
- Ранее на mobile элементы были смещены вправо примерно на `14.6px`; исправлено структурно, не через `margin-left`.

### Timeline / “Тайминг дня”

- В `src/components/Timeline.tsx` внутри `.timeline` есть декоративные элементы:
  - `.timeline__corner-art--top-left`
  - `.timeline__corner-art--bottom-right`
- Угловые картинки и линии должны двигаться вместе, как единая композиция.
- В конце `src/styles/limoncello.css` есть финальный блок:
  - `Italy timing final lock: corner art and frame are anchored to the same box`
- Не возвращать старые `section[aria-labelledby='timeline-title']::before/::after` для угловых картинок — они отключены.
- Для настройки композиции использовать CSS-переменные в `.timeline`, например:
  - `--timing-corner-w`
  - `--timing-tl-left`
  - `--timing-tl-top`
  - `--timing-br-right`
  - `--timing-br-bottom`
  - `--timing-frame-top-offset`
- Mobile-композиция была признана пользователем эталонной; desktop подгоняли под mobile.
- Иконки над событиями тайминга должны назначаться через `nth-of-type`, а не `nth-child`, потому что внутри `.timeline` есть декоративные `span`.
- Текущие соответствия:
  - 1: flowers
  - 2: rings
  - 3: glasses
  - 4: rings
  - 5 “Банкет”: dancers
  - 6 “Завершение банкета”: envelope
- Размер иконок сейчас увеличен:
  - desktop: `92px x 68px`
  - mobile: `70px x 52px`
